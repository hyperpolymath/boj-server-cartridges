== Overview

Linear and its related services, behind one cartridge: issues, comments,
projects, milestones, teams, cycles, labels, workflow states, users, documents,
initiatives and attachments — 27 MCP tools over the Linear GraphQL endpoint at
https://api.linear.app/graphql.

== Architecture

[cols="1,1,2"]
|===
| Layer | Language | Purpose

| ABI
| Idris2
| Formally verified connection state machine (`SafeComms.idr`) with
  dependent-type proofs for all valid transitions

| FFI
| Zig
| ADR-0006 five-symbol conformance layer: thread-safe session pool and the
  SafeComms state machine in C-ABI form. *No HTTP transport* — see below.

| Runtime
| Deno (`mod.js`)
| The working implementation. Calls the Linear GraphQL API and is what the
  BoJ loader executes.
|===

=== Which layer actually serves Linear

`mod.js` does. The Zig FFI deliberately answers `{"result":{"status":"stub"}}`
for every tool: it proves the ABI contract, not the API.

That is why `cartridge.json` leaves `available` unset (false). The Foundry
truthfulness probe (`tools/foundry/stages/harness.sh`) only passes this
cartridge *because* it does not claim to serve Linear over the FFI. Wiring a
real transport into the FFI is the prerequisite for ever setting
`available: true` — setting that flag while the FFI still stubs would be
precisely the over-claim the probe exists to catch.

== Authentication

Set `LINEAR_API_KEY` (sourced from `vault-mcp` in production).

[IMPORTANT]
====
Linear personal API keys (`lin_api_...`) go in the `Authorization` header
*raw, with no `Bearer` prefix*. `Bearer` is correct only for OAuth2 access
tokens. Sending `Bearer lin_api_...` fails with 401.

`mod.js` picks the right form automatically: keys beginning `lin_api_` are sent
raw, anything else is treated as an OAuth2 token and prefixed with `Bearer`.
====

== Rate limiting

Linear reports exhaustion as *HTTP 400* carrying a GraphQL error with
`extensions.code == "RATELIMITED"` — *not* HTTP 429. Code that only tests for
429 will misread a rate-limit as a generic bad request. `mod.js` detects the
`RATELIMITED` extension code and normalises it to a `429` result with
`rateLimited: true`, surfacing the `X-RateLimit-*` headers under `limits`.

Budgets: 5,000 requests/hour and 3,000,000 complexity points/hour per API key;
a single query may not exceed 10,000 points.

== Tools

[cols="1,3"]
|===
| Group | Tools

| Issues
| `linear_list_issues`, `linear_get_issue`, `linear_create_issue`,
  `linear_update_issue`, `linear_assign_issue`, `linear_set_priority`,
  `linear_move_to_project`, `linear_archive_issue`, `linear_search_issues`

| Comments
| `linear_list_comments`, `linear_create_comment`

| Projects
| `linear_list_projects`, `linear_get_project`, `linear_create_project`,
  `linear_update_project`, `linear_list_project_milestones`

| Teams & workflow
| `linear_list_teams`, `linear_get_team`, `linear_list_cycles`,
  `linear_list_labels`, `linear_list_workflow_states`

| Users
| `linear_list_users`, `linear_whoami`

| Docs & initiatives
| `linear_list_documents`, `linear_get_document`, `linear_list_initiatives`

| Attachments
| `linear_create_attachment`
|===

`linear_get_issue` accepts either a UUID or a human identifier such as
`ENG-123`; Linear's `issue(id:)` field only resolves UUIDs, so identifiers are
routed through a filtered `issues` query instead.

Priorities are Linear's own scale: `0` none, `1` urgent, `2` high, `3` medium,
`4` low.

== Usage

[source,sh]
----
# Sourced from vault-mcp in production. The value is a Linear personal API key;
# it is deliberately not written out here, even as an example — the estate's
# secret scanner treats a realistic-looking key prefix as a finding.
export LINEAR_API_KEY="$(vault-mcp read linear/api_key)"

# who am I?
deno run --allow-net --allow-env -e '
  import { handleTool } from "./mod.js";
  console.log(await handleTool("linear_whoami", {}));
'
----

== Tests

[source,sh]
----
tests/integration_test.sh    # FFI build + Zig unit tests + Idris2 ABI check
tests/parity_test.sh         # cartridge.json <-> mod.js <-> FFI tool-name parity
benchmarks/quick-bench.sh    # latency of the GraphQL transport
----

The tool list is declared in three places — `cartridge.json`, `mod.js` and the
`TOOLS` table in `ffi/linear_mcp_ffi.zig`. `parity_test.sh` fails if they drift.

== Toolchain

Pinned by `.tool-versions`: `zig 0.15.1`, `deno 2.7.14`, `idris2 0.8.0`. Zig
0.16 removed `std.meta.intToEnum` and `Compile.linkLibC`, both of which this
repo's Zig still uses, so the FFI does not build on 0.16.
