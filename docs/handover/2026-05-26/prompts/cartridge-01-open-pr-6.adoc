== Pitfalls

* *Do not retry `gh pr create` in a tight loop.* Hammers the
  secondary rate-limit further. One attempt, evaluate, wait if blocked.
* *Do not push more commits to the branch* expecting the PR to grow —
  any further work should wait until PR #6 lands so subsequent PRs branch
  from main cleanly.
* *Squash-merge changes the commit SHA.* Don't pile follow-ups onto this
  branch after the PR opens (see boj-server `.claude/CLAUDE.md` §PR
  Workflow).
