= proof-lsp — Proof Assistant Language Server cartridge

Scaffolded cartridge for a Language Server Protocol implementation covering
*Coq*, *Lean*, *Isabelle*, and *Agda*. Companion to `proof-mcp` (MCP slice of
the same domain).

== State

*Status:* `adapter-implemented` (per `cartridge.json#/states`).

The cartridge directory exists, the manifest declares the LSP capability
surface, the source lineage is archived under `_source-archive/`, and the
Deno adapter lives in `adapter/`. The Idris2 ABI (`abi/`), Zig FFI
(`ffi/`), and panels (`panels/`) are still empty — to be filled when this
cartridge promotes from `Ayo` to `Shield` tier.

== Source lineage

The original implementation is an Elixir LSP server living in the now-archived
`polystack/poly-proof/lsp/` (1552 LOC across 12 modules):

----
poly_proof_lsp.ex                       — top-level module (34 LOC)
poly_proof_lsp/application.ex           — OTP entry (22 LOC)
adapters/behaviour.ex                   — backend trait (99 LOC)
adapters/agda.ex                        — Agda adapter (150 LOC)
adapters/coq.ex                         — Coq adapter (150 LOC)
adapters/isabelle.ex                    — Isabelle adapter (152 LOC)
adapters/lean.ex                        — Lean adapter (144 LOC)
adapters/supervisor.ex                  — supervision tree (29 LOC)
lsp/server.ex                           — LSP framing/dispatch (222 LOC)
lsp/handlers/completion.ex              — completion handler (196 LOC)
lsp/handlers/diagnostics.ex             — diagnostics handler (180 LOC)
lsp/handlers/hover.ex                   — hover handler (174 LOC)
----

These files are preserved verbatim in `_source-archive/elixir-lib/` for the
port effort. They are not built by this cartridge.

== Port plan

. *adapter/server.ts* — DONE. Deno LSP framing + JSON-RPC dispatch.
. *adapter/handlers/* — DONE. All five handlers wired.
. *adapter/backends/* — DONE (skeletal). Subprocess wiring + simple output
  parsing for Coq/Lean/Agda; Isabelle is a PIDE-placeholder until an
  isabelle-server channel is added. Per-backend hover currently returns
  null; goal extraction + interactive tactic application are the next
  fidelity bumps (see `_source-archive/elixir-lib/lsp/handlers/hover.ex`
  + `adapters/{coq,lean,agda}.ex` `get_goals/3` for the reference logic).
. *ffi/zig/* — not started. Add only when measured slow.
. *abi/idris2/* — required for promotion from `Ayo` to `Shield` tier; not
  started.

== Why a separate cartridge for LSP

`proof-mcp` is consumed by AI agents (loads obligations, returns
verified/failed). `proof-lsp` is consumed by editors (returns diagnostics,
hovers, completions in real time as the user types).

Both wrap the same proof-assistant binaries but expose them through different
protocols, with different latency/UX requirements, and different state
lifecycles. Keeping them as separate cartridges:

* lets editor users avoid pulling the MCP slice and vice versa;
* lets the LSP cartridge be upgraded for editor responsiveness without
  destabilising agent workflows;
* matches the role-suffix taxonomy defined in
  https://github.com/hyperpolymath/standards/pull/200[standards#200].
