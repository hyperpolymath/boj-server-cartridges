== Suggested starting cartridge

*proof-lsp* — smallest surface area, 4 well-known backends (Coq/Lean/Isabelle/Agda), formally-verified-adjacent so a clean
testing target. The pattern proven here applies to the other 10.

== Concrete steps (per cartridge)

. Re-read `_source-archive/elixir-lib/poly_<name>_lsp.ex` and `lsp/server.ex` for the LSP framing.
. Re-read `_source-archive/elixir-lib/adapters/<backend>.ex` for each
   backend to learn its subprocess invocation, output parsing, and
   error-handling semantics.
. Create `adapter/`:
.. `server.ts` — LSP framing: read `Content-Length: <n>\r\n\r\n<json>` over stdio, dispatch by method, write framed response. Match the Phase 2 type style: `Result<T,E>`-returning helpers.
.. `handlers/initialize.ts` — return server capabilities (which LSP methods this cartridge supports).
.. `handlers/diagnostic.ts` — `textDocument/diagnostic` → backend.lint(uri) → diagnostics[]
.. `handlers/hover.ts` — `textDocument/hover` → backend.info(uri, position) → hover content
.. `handlers/completion.ts` — `textDocument/completion` → backend.complete(uri, position) → completion items
.. `handlers/executeCommand.ts` — `workspace/executeCommand` for cartridge-specific commands per the manifest
.. `backends/base.ts` — backend trait: `lint(uri)`, `info(uri, pos)`, `complete(uri, pos)`, `executeCommand(cmd, args)`.
.. `backends/<backend>.ts` — one file per backend, subprocess management for that backend's binary
. Create `adapter/deno.json` — pinned imports, test task (`deno test --allow-read --allow-run --allow-env`).
. Create `adapter/*_test.ts` — unit tests per handler + per backend stub. Smoke test driving an actual subprocess for at least one backend.
. Update manifest:
.. `states[]` add `adapter-implemented`.
.. (optional) Bump `version` to `0.2.0`.
. Delete the now-redundant `.gitkeep` in `adapter/`.
. Update `mod.js` `start()` to instantiate `adapter/server.ts` instead of throwing.

== Acceptance criteria

* `deno check adapter/server.ts adapter/handlers/*.ts adapter/backends/*.ts` passes.
* `deno test --allow-read --allow-run --allow-env` passes.
* Smoke test launches the adapter, sends `initialize`, receives server capabilities.
* Cartridge `mod.js` `start()` succeeds (was throwing — now boots the adapter).
* Manifest declares `adapter-implemented` in `states`.

== Pitfalls

* *Backend binary availability.* Tests must skip gracefully when the backend binary is absent. Use `Deno.which()` (or equivalent) and `t.skip()` on missing binaries.
* *LSP framing newlines.* `Content-Length` framing uses `\r\n` not `\n`. Get this wrong and the server hangs.
* *Per-backend output parsing varies enormously.* Don't try to abstract too early — each backend's stdout/stderr format is its own dialect. The Elixir behaviour module shows the *minimal* common interface.
* *Concurrency.* The Elixir version uses one GenServer per backend, supervised. The Deno port should use one Worker (or async task) per backend to preserve the isolation.
* *Auto-merge:* like every PR, set `--auto --squash --delete-branch` immediately. But throttle: don't try to file all 11 LSP adapter PRs in one burst (see secondary rate-limit memo).
